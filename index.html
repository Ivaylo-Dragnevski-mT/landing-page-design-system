02
Page Structure

Every page should follow this nesting order. Think of it as layers — each layer has one job and one job only. Never skip layers or combine their responsibilities.

.page-wrapper Outermost div on the page. Wraps everything. Use for overflow-x: hidden if needed. → Details below <main> .main-wrapper Wraps all content between the nav and footer. Signals "main content" to screen readers. → Details below <section> .section_hero One per content section. Name it descriptively: section_hero, section_features, section_testimonials… → Details below .padding-global Left/right page padding — and nothing else. Wraps the container. → Details below .container (or .container-small / .container-large) Centers content and sets max-width. Choose the size for the content type. → Go to Containers .padding-section-small / -medium / -large Top/bottom section spacing. Sits inside the container. → Go to Section Padding .section-content Wrapper for the section's actual content. Apply layout utilities (grid, flex, gaps) here. → Details below
Your content — headings, grids, cards, text… Everything else goes here: components, color classes, and content.
page-wrapper
One per page. Never style it heavily — it's an anchor, not a layout element.
Common use: add overflow-x: hidden directly on this element to prevent horizontal scroll issues site-wide.
main-wrapper
Use the <main> HTML tag. Excludes nav and footer.
In Droip: select the wrapper element between your nav and footer, change its HTML tag to <main> in the element settings, then add class main-wrapper.
section_[name]
Use the <section> HTML tag. Name each one uniquely.
section_hero section_features section_pricing section_testimonials section_faq section_cta

No styles live on these classes — they're for identification only. Apply backgrounds via bg-* classes.

padding-global
On its own div wrapping the container — nothing else on it. Controls left/right padding only.
Sets padding-left and padding-right to --container-padding (1.25rem). Change this one variable to update all page margins site-wide.
section-content
Wrapper inside padding-section-*. Holds the section's heading, grids, cards, and text.
A structural wrapper — like section_[name], it carries no required styles. Apply layout utilities here (grid, flex, gap-*) to arrange the content.

Spacing Utilities

Reference only — not in the master snippet. Combine a direction class with a size class on the same element. The direction defines which sides receive padding; the size sets the value via a scoped CSS custom property.

⚠️ These classes are a Droip-side utility — apply them in the page builder. They are not included in the CSS snippet pasted into Custom Code.
